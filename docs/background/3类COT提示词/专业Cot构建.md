# 博士论文与研究论文驱动的 10 类专业 CoT 构建
本文档给出一个具体方案：如何基于博士论文和研究论文，构建 10 类材料/化学领域专业 CoT 数据，并保存统一顶层思维链模板与 10 类专属思维链模板。
## 10类专业类别
1. 性能提升路径：是否有改性前后性能对比、短板描述和机制解释
2. 构效关系/结构-性能关系：是否有同一系列样品、控制变量和趋势数据
3. 候选分子/材料优选决策：是否有多个候选、评价指标和最终优选依据
4. 反事实结构改造：是否有可替换结构变量、相似对照样品或趋势证据
5. 失败原因诊断：是否有性能下降、失效样品、副作用或低性能负例
6. 多目标约束优化：是否有多个目标之间的冲突、折中设计和验证结果
7. 机理到设计策略迁移：是否有已验证机理、可调控变量和可迁移设计方向
8. 实验条件/工艺优化：是否有工艺参数扫描、性能趋势和最优窗口
9. 实验方案生成：是否有明确研究目标、可调变量、对照组和验证方法
10. 实验设计配方：是否有组分比例、浓度范围、配方矩阵或工艺窗口
## 文献到 CoT 的统一构建流程
本节给出从文献到 CoT 样本的统一加工流程。每一步都包含一个基础提示词，实际使用时再根据 10 类 CoT 的任务目标、证据类型和输出形态进行局部修改。每一类比的CoT提示词单独撰写。
### Step 1：筛选相关文献
基础提示词：

你是一名材料/化学领域 CoT 数据构建员。请阅读以下文献片段、图表说明或论文章节信息，判断其是否适合构建专业 CoT 样本。
输入材料：
<文献标题、摘要、结果讨论、实验部分、图表说明或博士论文章节内容>

请只基于输入材料中的证据完成以下判断：
1. 该文献是否适合构建 CoT 数据，给出 yes/no/partial。
2. 可支持的 CoT 类型，从以下 10 类中选择：性能提升路径、构效关系、候选优选决策、反事实结构改造、失败原因诊断、多目标约束优化、机理到设计策略迁移、实验条件/工艺优化、实验方案生成、实验设计配方。
3. 每个可构建类型对应的关键证据是什么。
4. 证据位于哪个章节、图、表或补充信息。
5. 哪些类型不建议构建，并说明证据缺口。

输出应简洁、可追溯，不要生成最终 CoT。

### Step 2：构建文献案例卡
请将以下文献内容整理成一张内部案例卡。案例卡用于后续生成 CoT 样本，不是最终训练输入。
输入材料：
<文献内容或已筛选片段>
请按照下面的字段定义抽取信息：
- source_id：文献或章节的唯一标识，用于内部追溯。
- source_type：来源类型，只能是 research_paper 或 phd_thesis。
- material_or_molecule：具体研究对象，不要只写宽泛类别。
- research_goal：当前实验或章节想解决的核心问题。
- baseline：基准样品、空白组、原始体系及其短板。
- modification_or_variable：被系统改变的结构、组成、配方、工艺或实验变量。
- control_samples：对照样品、系列样品、失败样品或候选对象。
- performance_metrics：性能指标、单位和测试条件。
- observed_results：实际观察到的结果、趋势和性能对比。
- mechanism_claim：由表征、计算或对照实验支撑的机制解释。
- process_conditions：制备、处理、测试或优化中的关键工艺条件。
- recipe_components：配方组分、前驱体、添加剂、溶剂、比例或浓度。
- validation_methods：用于验证结构、机制、性能或稳定性的方法及其验证对象。
- limitations：适用边界、未解决问题和结论限制。
- failure_or_risk：失败现象、副作用、负例或潜在风险。
- figures_or_tables：支撑信息所在的图、表、章节或补充信息位置。
要求：
1. 每个字段必须来自文献证据。
2. 无证据字段写 null 或 []，不要补写。
3. 保留关键数值、单位、实验条件和图表编号。
4. 区分事实结果、机制解释、工艺条件和配方组成，不要混写。
5. 不要把案例卡写成自然语言总结。

### Step 3：判定可构建的 CoT 类型
请根据以下案例卡判断可以构建哪些 CoT 类型，并为每一类给出构建理由和证据缺口。
输入案例卡：
<case_card>
请输出：
1. 可构建 CoT 类型列表。
2. 每一类 CoT 的任务目标。
3. 支撑该类型的文献证据。
4. 该类型可生成的样本数量建议：1 条、2-3 条、或多条。
5. 不可构建类型及原因。
判断规则：
- 至少有一个明确变量、一个目标指标和一条证据链，才可构建。
- 只有相关性证据时，不得写成确定因果。
- 只有单个最佳样品但没有对照时，不建议构建构效关系或工艺优化。

### Step 4：写成具体 input
请根据以下案例卡和指定 CoT 类型，生成一个适合作为训练样本 input 的具体专业问题。
输入：
case_card = <案例卡>
cot_type = <指定 CoT 类型>
要求：
1. input 必须像真实科研任务、实验观察、设计需求或诊断任务。
2. 不要出现“根据某文献”“本文报道”“作者发现”等文献来源表达。
3. 必须包含必要背景、关键变量和目标性能。
4. 不要把案例卡字段原样堆叠成 JSON。
5. 如果需要数值、单位或条件，必须与案例卡保持一致。
请生成 1 个主 input，并可额外给出 2 个不同问法的备选 input。

###  Step 5：按对应 CoT 模板生成 chainofThought
请根据以下 input、案例卡和 CoT 类型，生成专业 CoT 推理步骤。
输入：
input = <训练输入>
case_card = <案例卡>
cot_type = <指定 CoT 类型>
要求：
1. chainofThought 必须围绕 input 展开，不要复述文献背景。
2. 每一步都要服务于最终 output。
3. 推理中必须体现关键变量、性能指标、机制映射和证据-性能关联。
4. 不要引入案例卡没有支撑的结论。
5. 对不确定、条件依赖或只存在相关性的结论，要保留边界。
6. 不要在训练文本中反复出现 source_id 或文献名称；追溯信息放入 evidence_trace。
输出格式：
chainofThought = [
 "步骤 1 ...",
 "步骤 2 ...",
 "步骤 3 ..."
]

### Step 6：形成 output
请根据 input 和 chainofThought 生成最终 output。
输入：
input = <训练输入>
chainofThought = <推理步骤>
cot_type = <指定 CoT 类型>
要求：
1. output 必须直接回答 input。
2. 不要重复完整推理过程。
3. 结论应明确，但必须保留必要条件、适用边界和验证要求。
4. 不要给出文献没有支撑的确定性结论。
5. 输出应是可执行的解释、选择、预测、诊断、优化策略、实验方案或配方建议。

### Step 7：质检
请对以下 CoT 样本进行质检，判断是否可进入训练集。
输入：
sample = {
 "cot_type": "...",
 "input": "...",
 "chainofThought": [...],
 "output": "...",
 "evidence_trace": {...}
}
请检查：
1. input 是否是清楚的具体问题。
2. chainofThought 是否逐步支撑 output。
3. output 是否直接回答 input。
4. 是否存在文献证据不足或过度外推。
5. 是否保留实验条件、适用边界和必要验证。
6. 单位、指标、条件是否一致。
7. 是否把相关性误写成因果性。
请输出：
- quality_decision: pass / revise / reject
- main_issues: [...]
- revision_suggestions: [...]
- evidence_risk: low / medium / high

### Step 8：输出结果
请根据质检结果，输出最终的 CoT 样本。
输入：
sample = {
 "cot_type": "...",
 "input": "...",
 "chainofThought": [...],
 "output": "...",
 "evidence_trace": {...}
}
请输出：
- final_sample: {...}
