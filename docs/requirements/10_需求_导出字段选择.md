# 10_需求_导出字段选择

## 背景
当前下载和合并下载直接导出全量 JSON，无法控制导出哪些字段。用户希望导出时只保留需要的字段，减少冗余数据。

## 功能描述

### 下载流程变更
原流程：点击下载 → 直接下载全量 JSON
新流程：点击下载 → 弹出字段选择 Dialog → 用户勾选字段 → 确认后下载仅含勾选字段的 JSON

### 合并下载流程变更
原流程：多选 → 点击合并下载 → 确认弹窗 → 直接下载合并全量 JSON
新流程：多选 → 点击合并下载 → 弹出字段选择 Dialog → 用户勾选字段 → 确认后下载仅含勾选字段的合并 JSON

### 字段选择 Dialog
- **字段来源**：从已有预览数据中解析（取几条记录即可提取所有字段名）
- **字段展示**：
  - 顶层字段一组
  - `extra` 子字段一组（如 extra.options、extra.confidence、extra.reason）
- **默认勾选**（忽略大小写匹配）：
  id, domain, category, task_type, input, output, cot, corpus_cate, scene, source, source_id, originContent, source_type, knowledge, difficulty, Relevance, Clarity, Scientific, Reasoning, Terminology, score
- **匹配规则**：默认勾选列表中的字段名与实际数据字段名匹配时忽略大小写（如数据中是 `relevance`，列表中是 `Relevance`，仍默认勾选）
- **extra 处理**：如果勾选了部分 extra 子字段但未勾选全部，导出时 `extra` 对象只保留勾选的子字段；如果没勾选任何 extra 子字段，导出时不保留 `extra` 字段本身

### 导出结果
- 仅包含勾选的字段，未勾选的字段剔除
- extra 对象仅包含勾选的子字段
- JSON 格式不变，其余结构保持一致

- `backend/app/routers/file_manage.py`：下载接口 `/file-manage/download/{id}` 和合并下载接口 `/file-manage/merge-download` 新增可选 `fields` 参数
- `backend/app/services/file_manage_service.py`：新增字段过滤逻辑

- `frontend/src/api/index.js`：下载和合并下载接口新增 fields 参数
