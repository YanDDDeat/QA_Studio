# 需求：问题生成前文本预处理

## 背景

QA Studio 的 6 阶段 Pipeline 中，**只有 `question_generate`（阶段 1）直接消费用户上传 JSON 中的原始 `text` 文本**——其余阶段都是基于阶段 1 写入数据库的字段（`input`/`output`/`originContent` 等）继续处理。

上传的 JSON 通常来源于「文献/图书 PDF → markdown → chunk 切分」这条流水线，因此原始 chunk 中往往混杂大量**无意义内容**：

- **目录页**：`第一章 ..... 1`、`1.1 研究背景 ..... 3` 这类页码目录
- **参考文献**：`[1] He K, et al. ...`、DOI、期刊缩写堆积的段落
- **图片链接为主**：`![](url)` 占主体，正文只有「如图所示」一句
- **表格残骸**：md 表格语法 `|---|---|` 占大头，自然语言比例极低
- **过短 chunk**：标题、`本章小结`、孤立的页眉页脚等几个字
- **重复页眉/页脚**：每页都重复出现的书名、页码模板

把这些垃圾喂给 LLM 不仅浪费 token，还会产出大量低质量问答对，污染后续 5 个阶段的数据。

## 目标

在 `question_generate` 阶段调 LLM **之前**，插入一个**纯函数预处理层**，对原始 chunks 完成：

1. **清洗**：剥离图片链接、md 表格语法、识别并去除高频页眉页脚、折叠多余空行
2. **过滤**：识别 5 类无意义 chunk（目录 / 参考文献 / 图片堆 / 表格残骸 / 低自然语言），直接跳过
3. **合并**：对过短（< 50 token）的 chunk，按顺序向后吸收邻居直到达到阈值
4. **输出**：把跳过的 chunk 单独写到一份过滤文件，方便用户审查；统计写入任务日志

**核心约束：不动数据库、不动其他阶段、不动前端**。预处理结果不持久化，每次任务启动都重新跑一次（纯函数，确定性保证）。

```python
def preprocess_chunks(
    raw_records: list,             # 上传 JSON 解析后的原始记录列表
    text_field: str,                # 取文本的字段名（如 "text"）
) -> tuple[list[ProcessedChunk], PreprocessStats]:
    """对原始 chunks 做清洗 + 过滤 + 合并。

    Returns:
        processed_chunks: 实际要送 LLM 的 chunk 列表（每条带清洗后文本和元信息）
        stats: 统计信息（保留/跳过分布、被跳过 chunk 的明细列表）
    """
```

#### 数据结构

```python
@dataclass
class ProcessedChunk:
    text: str                       # 清洗+合并后的最终文本
    source: str                     # 来源（沿用原 record 的 source）
    source_id: str                  # 来源 ID（合并时取第一个被合并的）
    merged_from: list[int] | None   # 合并的原始 index 列表，单条则为 None
    original_record: dict           # 原始 record（供主循环继续读 source/source_id 等字段）

@dataclass
class SkippedChunk:
    original_index: int             # 原始 JSON 中的下标
    text: str                       # 原始文本（未清洗，方便审查）
    skip_reason: str                # 见下方枚举

@dataclass
class PreprocessStats:
    original_count: int             # 原始 chunk 数
    kept_count: int                 # 保留数（含合并产生的）
    kept_by_merge_count: int        # 其中由合并产生的数量
    skipped_count: int              # 跳过总数
    skip_breakdown: dict[str, int]  # 按 reason 分类的数量
    skipped_records: list[SkippedChunk]  # 跳过明细（用于写过滤文件）
    header_footer_blacklist: list[str]   # 识别出的页眉页脚（仅日志展示）
```

#### `skip_reason` 枚举（7 类，统一中文）

| reason | 触发条件 |
|---|---|
| `内容为空` | 原始文本为空 / 取不到 text_field |
| `目录` | 章节模式行（第X章/节、X.X、中文数字「一/二/三、」）≥ 2 行 + 占比 > 30% + 至少 1 个页码标记（点引导 `...123` 或括号页码 `(123)`）；或短行占比 > 70% 且 ≥ 5 行且含页码标记 |
| `参考文献` | `^\[\d+\]` 行首占比 > 30% 或 DOI/期刊缩写关键词密集 |
| `图片为主` | 图片链接占字符比例 > 50% 且剥离后长度 < 100 token |
| `表格残骸` | 剥离 md 表格语法后自然语言（中英文字符）占比 < 60% |
| `低自然语言` | 中英文字符占比 < 60%（剥离图片/表格后判定） |
| `末尾过短` | 长度 < 100 token 且无后续 chunk 可吸收（末尾孤儿） |
| `已合并` | 被其他 chunk 吸收（不算"过滤"，但要在统计里展示并写入过滤文件，标注 `merged_into_index`） |

#### 默认阈值（写为模块级常量）

```python
MIN_TOKEN_THRESHOLD = 100           # too_short 触发合并阈值
SHORT_LINE_LEN = 30                 # 行长 < 此值算"短行"
TOC_SHORT_LINE_RATIO = 0.70         # 短行占比 > 此值判定为目录
REFERENCES_LINE_RATIO = 0.30        # 参考文献行占比 > 此值判定
IMAGE_CHAR_RATIO = 0.50             # 图片链接占字符比例 > 此值
LOW_ALPHA_RATIO = 0.60              # 自然语言占比 < 此值判定低质
HEADER_FOOTER_MIN_FREQ = 0.30       # 行重复出现率 > 此值视为页眉页脚
HEADER_FOOTER_MAX_LEN = 40          # 仅短行参与页眉页脚识别
MERGE_SEPARATOR = "\n\n"            # 合并 chunk 时使用的分隔符
```

#### 处理流程（按顺序执行）

```
preprocess_chunks(raw_records, text_field):
    # ── 步骤 1：抽取原始文本 ──
    texts = [extract_text(r, text_field) for r in raw_records]
    
    # ── 步骤 2：预扫描页眉页脚 ──
    blacklist = detect_headers_footers(texts)
    
    # ── 步骤 3：逐条清洗 + 分类 ──
    processed_items = []  # 每项: (cleaned_text, skip_reason | None)
    for text in texts:
        if not text:
            processed_items.append((text, "empty"))
            continue
        cleaned = clean_text(text, blacklist)
        reason = classify(cleaned, raw=text)   # too_short 时返回 None 让步骤 4 处理
        processed_items.append((cleaned, reason))
    
    # ── 步骤 4：合并 too_short ──
    final_chunks = []
    skipped = []
    i = 0
    while i < len(processed_items):
        text, reason = processed_items[i]
        if reason is None and estimate_tokens(text) >= MIN_TOKEN_THRESHOLD:
            # 直接通过
            final_chunks.append(make_chunk(text, raw_records[i], merged_from=None))
            i += 1
        elif reason is None and estimate_tokens(text) < MIN_TOKEN_THRESHOLD:
            # 触发合并：向后吸收，同 source 且 reason 也为 None
            merged_text, merged_indices, end = merge_forward(processed_items, raw_records, i)
            if estimate_tokens(merged_text) >= MIN_TOKEN_THRESHOLD:
                final_chunks.append(make_chunk(merged_text, raw_records[i],
                                               merged_from=merged_indices))
                # 把被吸收的标记为 merged
                for idx in merged_indices[1:]:
                    skipped.append(SkippedChunk(idx, texts[idx], "merged"))
                i = end
            else:
                # 末尾孤儿
                skipped.append(SkippedChunk(i, texts[i], "too_short_at_end"))
                i += 1
        else:
            skipped.append(SkippedChunk(i, texts[i], reason))
            i += 1
    
    stats = compute_stats(final_chunks, skipped, blacklist)
    return final_chunks, stats
```

#### Token 粗估公式

```python
def estimate_tokens(text: str) -> int:
    """中文字数 × 1.5 + 英文单词数 × 1.3 + 其他字符 × 0.5
    
    针对 qwen 类 tokenizer 的粗略估算，零依赖。
    """
    import re
    chinese = sum(1 for c in text if '一' <= c <= '鿿')
    english_words = len(re.findall(r'[a-zA-Z]+', text))
    english_chars = sum(len(w) for w in re.findall(r'[a-zA-Z]+', text))
    other = max(0, len(text) - chinese - english_chars)
    return int(chinese * 1.5 + english_words * 1.3 + other * 0.5)
```

#### 页眉页脚识别

```python
def detect_headers_footers(texts: list[str]) -> list[str]:
    """统计所有 chunk 中的短行（<= 40 字）出现频率，
    出现率 > 30% 的行视为页眉/页脚，加入黑名单。"""
    from collections import Counter
    line_counter = Counter()
    total = len(texts) or 1
    for text in texts:
        seen_in_this_chunk = set()
        for line in text.splitlines():
            line = line.strip()
            if 0 < len(line) <= HEADER_FOOTER_MAX_LEN and line not in seen_in_this_chunk:
                line_counter[line] += 1
                seen_in_this_chunk.add(line)
    return [line for line, count in line_counter.items()
            if count / total >= HEADER_FOOTER_MIN_FREQ]
```

#### `originContent` 写入

写入 Dataset 时，`originContent = text_content`（清洗后的版本），后续阶段（answer_generate / data_evaluate）拿到的也是干净文本。

#### 中断恢复

`start_index > 0` 时（暂停恢复 / 进程重启）：
- 依然重新读 JSON、重新跑预处理
- **不重写过滤文件**（`start_index == 0` 时才写）
- 进度索引 `start_index` 仍然基于预处理后的 `processed_chunks` 计算

**确定性保证**：所有阈值是模块常量，规则是固定函数，输入相同则输出必然相同。若中途代码阈值被修改，旧任务恢复时可能错位——本次需求**接受此风险**（视同 prompt/代码升级，发版前评估）。

## 不涉及

- ❌ 数据库：不动任何表
- ❌ 其他 5 个阶段（knowledge_generate / question_validate / answer_generate / answer_validate / data_evaluate）：不动
- ❌ 前端：不改 UI；task_logs 走现有日志展示通道自动生效；过滤文件通过现有文件管理列表自动可见可下载
- ❌ 数据中心 / 失败文件机制：不动
- ❌ Prompt 配置 / 阈值前端可调入口：本次写死代码常量，未来如需调整再加入口

## 兼容性

- **存量任务**：已跑完的旧任务不受影响；新跑的任务才走预处理
- **暂停/恢复**：基于"预处理是确定性"假设，恢复时重跑预处理结果一致
- **JSON 单对象输入**：现有 `data = [data]` 兼容已生效，预处理同样支持
- **空 text_field 记录**：预处理统一打 `empty` 标记并跳过，不再依赖主循环里的 `if not text_content: continue` 分支（该分支保留作为冗余防御）

## 测试方案

### 单元测试（`tests/test_preprocess_service.py`，新建）

针对 `preprocess_service.py` 的纯函数特性，写如下用例：

1. **Token 估算**：纯中文 / 纯英文 / 混合 / 空字符串
2. **清洗**：
   - 剥离 `![alt](url)` 后剩余文本正确
   - 剥离 md 表格分隔线 `|---|---|`
   - 命中页眉黑名单的行被删除
   - 多余空行折叠
3. **分类**：每类 `skip_reason` 各构造 1-2 个正例 + 1 个反例（KEEP）
4. **页眉识别**：构造 10 条 chunk，其中 5 条含同一短行 → 应识别为页眉
5. **合并算法**：
   - 短 chunk 吸收下一条 → 合并成功
   - 连续多个短 chunk 链式合并直到达标
   - 末尾孤儿短 chunk → `too_short_at_end`
   - 跨 source 合并：当前实现"按顺序"合并，不强制 source 检查（chunks 是按 source 顺序读入的，跨 source 极少且无害；如确需保护，在 merge_forward 内加 source 比对）
6. **整体集成**：构造一份 50 条混合脏数据 + 干净数据的 JSON，断言保留数 / 跳过分布
