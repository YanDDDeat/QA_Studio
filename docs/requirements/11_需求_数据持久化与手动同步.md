# 需求：数据持久化与手动同步

## 背景

当前 Pipeline 任务执行时，Dataset 数据逐条写入 DB（每条记录 `db.commit()`），但磁盘 JSON 文件只在任务**全部完成**后才调用 `write_datasets_to_file()` 刷新一次。如果任务中途暂停或异常退出，DB 中已有部分数据，但磁盘 JSON 文件仍为空数组 `[]`。

数据中心（DataManage.vue）的文件预览直接从磁盘 JSON 文件读取内容，而非从 DB，因此中途暂停的任务在数据管理页面查看时显示为空。

此外，目前没有手动触发"DB → 磁盘文件"同步的手段，用户无法在 DB 数据变更后主动让文件保持一致。

## 需求1：任务暂停/停止时自动刷盘

### 规则
- 在每个 Pipeline 阶段的 `_run_*_task()` 协程中，当检测到任务状态为 `PAUSED` 准备退出时，**先调用 `write_datasets_to_file(db, output_file.id)`**，将已处理的 Dataset 数据写入磁盘 JSON 文件，再 `return`
- 同样适用于因连续失败 20 次导致任务标记 `FAILED` 退出的场景：退出前也刷盘
- 刷盘逻辑为从 DB 全量读取该 file_id 下所有 Dataset → 序列化为 JSON → 整体写入文件（覆盖写）

### 涉及文件
- `backend/app/routers/question_generate.py` — `_run_question_generate_task()`
- `backend/app/routers/knowledge_generate.py` — `_run_knowledge_generate_task()`
- `backend/app/routers/question_validate.py` — `_run_question_validate_task()`
- `backend/app/routers/answer_generate.py` — `_run_answer_generate_task()`
- `backend/app/routers/answer_validate.py` — `_run_answer_validate_task()`
- `backend/app/routers/data_evaluate.py` — `_run_data_evaluate_task()`
- `backend/app/routers/dataset_assessment.py` — `_run_dataset_assessment_task()`

### 修改位置
- 在 PAUSED 检测后 `return` 前，插入 `write_datasets_to_file(db, output_file.id)` 调用
- 在连续失败 `FAILED` 退出前，插入同样的刷盘调用
- 异常 `except` 块中，在标记 `FAILED` 后也尝试刷盘（如果 output_file 已创建）

## 需求2：数据中心文件列表新增"同步到文件"按钮

### 规则
- 在 DataManage.vue 文件列表的操作列（目前有查看、下载），新增一个"同步到文件"按钮
- 点击后调用后端接口 `POST /file-manage/sync/{file_id}`
- 后端执行 `write_datasets_to_file(db, file_id)`，从 DB 全量重写磁盘 JSON 文件
- 返回同步结果：同步了多少条记录
- 前端收到成功响应后提示用户，并刷新文件预览内容

### 后端接口
- **路径**: `POST /file-manage/sync/{file_id}`
- **权限**: 当前用户（或 admin）只能同步自己的文件
- **逻辑**:
  1. 查询 File 记录，校验存在性和权限
  2. 查询该 file_id 下 Dataset 记录数量
  3. 调用 `write_datasets_to_file(db, file_id)` 全量覆盖写入
  4. 返回 `{ "synced_count": N, "filename": "xxx.json" }`
- **错误处理**: file_id 不存在 → 404；非 JSON 文件 → 400

### 涉及文件
- `backend/app/routers/file_manage.py` — 新增 `/sync/{file_id}` 接口
- `frontend/src/views/DataManage.vue` — 操作列新增"同步到文件"按钮
- `frontend/src/api/index.js` — 新增 `syncFileToDisk(file_id)` API 函数

### UI 变更
- 文件列表操作列增加一个按钮，图标用刷新/同步图标（如 `Refresh`），文字"同步"
- 点击后弹出确认提示："确定要将数据库中的数据同步到文件吗？这将覆盖文件当前内容。"
- 同步过程中按钮显示 loading 状态
- 成功后提示"已同步 N 条记录到文件"，并自动刷新预览
