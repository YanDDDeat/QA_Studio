# 需求：JSON 文件合并工具（前端独立页面）

## 背景

业务侧经常需要将多个来源不同、字段不完全一致的 JSON 数据文件合并成一个统一字段的数据集。之前的 `08_需求_JSON文件合并导出.md` 是基于「文件管理」页面已上传数据集的合并下载，字段直接 union（保留所有字段）；本需求面向**任意外部 JSON 文件**的本地合并，强调字段一致性校验和「保留字段交集」的语义，是独立工具页面。

与 08 的区别：

| 维度 | 08（已实现） | 本需求 |
|---|---|---|
| 数据来源 | 系统内文件管理已上传的 dataset | 用户从本机选择的任意 JSON 文件 |
| 字段策略 | 直接合并所有字段（union） | 取字段名**交集**后裁剪 |
| 必填校验 | 无 | 每条记录必须含 `source_id` / `source` / `source_type` |
| 实现 | 后端接口 | 纯前端（浏览器内合并下载） |
| 入口 | 文件管理页 | 独立「工具」菜单 |

## 目标

提供一个独立的工具页面，让用户上传多个 JSON 文件，按统一规则校验和合并后下载结果，不经过后端、不写入数据库。

## 范围

### 页面与路由
- 新页面：`frontend/src/views/JsonMergeTool.vue`
- 路由：`/json-merge-tool`，meta title「JSON 文件合并工具」
- 菜单入口：`Layout.vue` 新增一级分组「工具」（图标 `Tools` 或 `Operation`），下挂此页面

### 输入约束
- 文件类型：仅接受 `.json`
- 文件内容：必须是 JSON **数组** `[{...}, {...}, ...]`，单个对象或其他类型一律视为不合格
- 每条记录必须包含三个必填字段（同时存在且值非 null/undefined/空字符串）：
  - `source_id`
  - `source`
  - `source_type`
- 上传文件数量：≥ 2 个才能触发合并

### 校验失败处理
**严格模式**：只要任意文件、任意记录有以下任一问题，整次合并任务直接失败，UI 给出明确报错（哪个文件、第几条、缺什么字段），不允许部分合并：
- 文件不是合法 JSON
- 文件根不是数组
- 数组中存在元素不是对象
- 任一对象缺少 / 为空 `source_id` / `source` / `source_type`

用户需要自行处理坏文件后重新点合并。

### 合并规则
1. **字段交集**：扫描所有上传文件的所有记录，取每条记录字段名的并集 → 再对所有文件求交集，得到「公共字段集」。`source_id` / `source` / `source_type` 必然在交集里（已经过校验）。
2. **记录拼接**：按上传顺序，把所有文件的记录依次追加到结果数组，**不去重**。
3. **字段裁剪**：结果数组中每条记录只保留公共字段集中的键，其余字段丢弃。
4. **字段顺序**：按第一个文件中第一条记录的字段顺序排列（保留可读性）。

### UI 设计
- 标题区：「JSON 文件合并工具」+ 一行说明（"上传多个 JSON 数组文件，自动校验必填字段并按字段交集合并下载。"）
- 上传区：Element Plus `el-upload` 拖拽上传，支持多选、支持反复追加
- 文件列表表格（按上传顺序）：列含「文件名 / 大小 / 记录数 / 校验状态（OK / 错误简要）/ 操作（移除）」
- 统计提示区：在「合并并下载」按钮上方实时显示：
  - 当前文件数 N
  - 合计记录数（所有合格文件相加，校验通过后才显示）
  - 公共字段数 M 及字段名列表（校验通过后才显示）
- 「合并并下载」按钮：≥ 2 个文件时启用；点击后执行校验 → 校验失败弹出 `ElMessage.error` 详细报错 → 校验通过则在浏览器生成 JSON 文件触发下载
- 输出文件名：`merged_{N}files_{YYYYMMDD_HHmmss}.json`

### 实现要点
- 纯前端：`FileReader` 读取每个文件 `text()`，`JSON.parse` 解析
- 大文件保护：单文件 > 50MB 直接拒收并提示
- 合并后用 `Blob` + `URL.createObjectURL` + `<a download>` 触发下载，结束后 `revokeObjectURL`
- 移除文件后实时重新统计

## 后端

无改动。不涉及任何接口、不涉及数据库。

## 测试方案

人工：
1. 上传 2 个合格 JSON（数组、含必填字段、字段部分重叠）→ 提示公共字段、合计记录数 → 下载校对内容
2. 上传含缺失字段记录的文件 → 报错指明文件名 + 行号
3. 上传非数组 JSON（单对象 / 字符串）→ 报错
4. 上传非 JSON 文件 → 上传环节就拒收
5. 单文件超 50MB → 拒收
6. 上传 1 个文件 → 「合并并下载」按钮 disabled
7. 三个文件字段完全无交集（除三个必填字段）→ 公共字段为 `source_id` / `source` / `source_type`，结果裁剪到只剩这三个字段

## 不做的事

- 不涉及任何后端接口
- 不写入数据库、不持久化合并历史
- 不做按 source_id 去重（明确不要）
- 不做字段类型一致性校验（仅校验字段名是否存在）
- 不接入用户认证业务流（独立工具，登录后即可使用）
- 不与文件管理模块集成（与 08 的合并下载并存，互不影响）
