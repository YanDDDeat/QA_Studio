# 需求 29：标注流水线2提示词模板管理

## 背景

当前「标注流水线2：专业 CoT 构建」使用 `docs/background/3类COT提示词/` 下的 Markdown 文件作为提示词来源：

- `专业Cot构建.md` 中的 Step 1 / Step 2 / Step 3 通用提示词。
- 10 类专业 CoT 子目录中的 Step 4 / Step 5 / Step 6 专属提示词。

这种方式适合作为系统默认模板，但普通使用者无法在页面中查看、复制、编辑或选择提示词版本。为了让使用者能在 COT/H-COT 模块内部管理标注流水线2的提示词，同时避免耦合全局配置中心或其他 Pipeline，本需求新增「标注流水线2提示词模板管理」能力。

本需求只面向「标注流水线2」，不改造全局 Prompt 管理，不影响问题生成、答案生成、通用生成等其他阶段。

## 目标

1. 在 COT/H-COT 模块下提供标注流水线2专属提示词管理入口。
2. 系统默认提示词作为共享只读模板，所有用户可用，但普通用户不可直接修改。
3. 用户需要修改提示词时，先复制系统模板或自己的历史模板，生成该用户独立的模板版本。
4. 新建标注流水线2任务时，用户选择一个完整的「提示词模板包版本」，而不是逐条选择 33 个 Prompt。
5. 每次运行开始时保存实际使用提示词快照，确保历史运行可追溯。
6. 不接入全局配置中心，不复用其他 Pipeline 的 Prompt 表和管理页面。

## 范围

### 包含

- COT/H-COT 分组下新增提示词管理入口或在标注流水线2页面中新增「提示词模板管理」按钮。
- 标注流水线2专属模板包列表。
- 模板包复制、重命名、设置为我的默认、删除未使用模板。
- 单个 Prompt 的查看和编辑。
- 系统默认模板只读展示。
- 新建标注流水线2任务时选择模板包版本。
- 运行时按模板包读取提示词。
- 运行目录保存提示词快照。

### 不包含

- 不接入全局配置中心 Prompt 管理。
- 不改造已有 6 阶段 Pipeline 的提示词管理逻辑。
- 不支持管理员审批发布流程。
- 不做复杂团队共享模板市场。
- 第一版不做 Prompt 在线试运行。
- 第一版不做多人协同编辑锁。

## 核心概念

### 提示词模板包

标注流水线2实际需要一整套提示词，不能让用户在新建任务时分别选择每个 Step 的 Prompt。因此需要将全部提示词作为一个「模板包」管理。

模板包包含：

```text
标注流水线2提示词模板包
├─ 通用步骤
│  ├─ Step 1：筛选相关文献
│  ├─ Step 2：构建文献案例卡
│  └─ Step 3：判定 CoT 类型
│
└─ 10 类 CoT 专属步骤
   ├─ 性能提升路径 CoT
   │  ├─ Step 4：生成 input
   │  ├─ Step 5：生成 chainofThought
   │  └─ Step 6：生成 output
   ├─ 构效关系-结构性能关系 CoT
   │  ├─ Step 4
   │  ├─ Step 5
   │  └─ Step 6
   └─ ...
```

当前共计：

- 通用 Step 1-3：3 份。
- 10 类 CoT × Step 4/5/6：30 份。
- 合计 33 份 Prompt。

页面和任务创建流程应按「模板包」管理，不应把 33 份 Prompt 平铺给用户选择。

### 系统默认模板

系统默认模板来自当前仓库内置提示词：

```text
docs/background/3类COT提示词/
```

系统默认模板规则：

- 所有用户可见。
- 所有用户可在新建任务时选择使用。
- 普通用户不可直接修改。
- 可作为用户模板的复制来源。
- 当用户模板不存在或读取失败时，可作为 fallback。

### 用户独立模板

用户若要修改提示词，必须先复制一个模板包：

```text
系统默认模板 v1
  ↓ 复制
用户A / 我的模板 v1
```

用户模板规则：

- 只对创建者可见、可编辑、可用于新建任务。
- 不影响系统默认模板。
- 不影响其他用户模板。
- 可设置为该用户的新建任务默认模板。
- 可从系统默认模板或自己的历史模板复制生成。

## 新建任务变更

「标注流水线2」新建任务弹窗新增字段：

```text
提示词模板版本：[系统默认模板 v1 ▼]
```

下拉分组：

```text
系统模板
- 系统默认模板 v1

我的模板
- 我的模板 v1
- 我的模板 v2
```

默认选择规则：

1. 若用户设置了「我的默认模板」，默认选该模板。
2. 否则默认选系统默认模板。
3. 可选优化：记住用户上一次选择的模板。

任务创建后，该任务绑定一个完整模板包版本。任务执行过程中不允许切换模板。

## 运行时规则

### 与需求 28：批量文献 JSON 输入的配合

需求 28「标注流水线2支持批量文献 JSON 输入」允许一个 run 内处理多篇文献。本需求与其配合时遵循以下规则：

1. 提示词模板选择是 run 级别配置，不是文献级别配置。
2. 一个批量 run 只绑定一个完整模板包版本。
3. 批量 run 创建成功后，只生成一次完整提示词快照，不为每篇文献重复创建快照。
4. 批量 run 内所有文献都从同一个 `prompts/` 快照目录读取 Prompt。
5. 用户在任务运行中修改模板，不影响当前批量 run 已处理或尚未处理的文献。
6. 批量 run 中不同文献可能在 Step 3 被判定为不同 CoT 类型；Step 4/5/6 应根据每篇文献自己的 Step 3 判定结果，从同一个模板包快照中读取对应 CoT 类型的专属 Prompt。
7. 历史 run 展示模板信息时，无论该 run 是单篇还是多篇，均展示 run 级模板信息和快照信息。
8. 如果批量 run 的某篇文献失败，不影响 run 级提示词快照，也不重新选择模板。

批量 run 的提示词生命周期：

```text
用户选择模板包版本
  ↓
创建 run 并绑定 prompt_template_id
  ↓
复制完整模板包到 run/prompts/ 快照目录
  ↓
逐篇处理文献，所有文献读取同一个 run/prompts/
  ↓
历史详情优先展示 run/prompts/ 快照，而不是当前模板内容
```

### 读取优先级

运行标注流水线2时，提示词读取优先级为：

```text
任务指定的用户模板包
  ↓ 若不存在或无权限
用户默认模板包
  ↓ 若不存在
系统默认模板包
  ↓ 若未初始化
仓库 docs/background/3类COT提示词/ 下的默认 md 文件
```

正常情况下，任务创建时必须明确写入 `prompt_template_id`，后续运行只读取该模板包。

### 快照规则

每次 run 创建成功后，必须保存本次实际使用的完整提示词快照：

```text
storage/professional_cot_runs/<run_id>/prompts/
├─ manifest.json
├─ common/
│  ├─ step1_screening.md
│  ├─ step2_case_card.md
│  └─ step3_target_judgement.md
└─ cot_types/
   ├─ performance_improvement/
   │  ├─ step4_input.md
   │  ├─ step5_chain.md
   │  └─ step6_output.md
   ├─ structure_property/
   │  ├─ step4_input.md
   │  ├─ step5_chain.md
   │  └─ step6_output.md
   └─ ...
```

运行详情页应展示：

```text
提示词模板：我的模板 v2
模板来源：用户模板
快照路径：prompts/
```

历史任务查看时，应优先展示快照内容，而不是当前模板内容。

第一版推荐使用标注流水线2专属文件化存储，不使用全局 Prompt 表。


## 接口设计

接口挂在标注流水线2专属路由下：

```text
/api/professional-cot/prompts/...
```

不挂到全局配置中心。

### 模板列表

```text
GET /api/professional-cot/prompts/templates
```

返回当前用户可用模板：

- 系统默认模板。
- 当前用户自己的模板。

### 获取模板详情

```text
GET /api/professional-cot/prompts/templates/{template_id}
```

返回：

- manifest。
- 左侧树结构。
- 当前选中 Prompt 内容，或全部 Prompt 元信息。

### 复制模板

```text
POST /api/professional-cot/prompts/templates/{template_id}/duplicate
```

请求：

```json
{
  "name": "我的模板 v1"
}
```

规则：

- 系统模板和用户自己的模板都可以复制。
- 复制结果归当前用户所有。

### 更新模板名称

```text
PUT /api/professional-cot/prompts/templates/{template_id}
```

仅允许修改当前用户自己的模板。

### 获取单个 Prompt

```text
GET /api/professional-cot/prompts/templates/{template_id}/items/{prompt_key}
```

`prompt_key` 示例：

```text
common.step1
common.step2
common.step3
performance_improvement.step4
performance_improvement.step5
performance_improvement.step6
structure_property.step4
```

### 保存单个 Prompt

```text
PUT /api/professional-cot/prompts/templates/{template_id}/items/{prompt_key}
```

请求：

```json
{
  "content": "新的提示词内容"
}
```

规则：

- 系统模板不可保存。
- 只能保存当前用户自己的模板。
- 内容不能为空。
- 保存后更新模板 `updated_at`。

### 恢复单个 Prompt 默认内容

```text
POST /api/professional-cot/prompts/templates/{template_id}/items/{prompt_key}/restore-default
```

规则：

- 仅用户模板可操作。
- 从系统默认模板中读取同 `prompt_key` 的内容覆盖当前项。

### 设置我的默认模板

```text
POST /api/professional-cot/prompts/templates/{template_id}/set-default
```

规则：

- 系统模板和当前用户自己的模板均可设为默认。
- 写入当前用户 `preferences.json`。

### 删除模板

```text
DELETE /api/professional-cot/prompts/templates/{template_id}
```

规则：

- 只能删除当前用户自己的模板。
- 系统模板不可删除。
- 已被历史 run 使用的模板第一版建议禁止删除，只允许归档或隐藏。

## 权限规则

1. 登录用户可查看系统默认模板。
2. 登录用户可复制系统默认模板为自己的模板。
3. 登录用户只能查看、编辑、删除自己的模板。
4. 普通用户不可修改系统默认模板。
5. 管理员第一版也不需要在页面修改系统默认模板；系统默认模板仍以仓库 md 文件为准。
6. 历史运行详情只能查看自己 run 的提示词快照，管理员是否可跨用户查看沿用现有任务权限规则。

## 编辑与版本规则

第一版可采用较轻量规则：

- 用户模板允许直接编辑。
- 每次 run 都保存提示词快照，因此历史运行不会被当前模板修改影响。
- 模板的 `updated_at` 体现当前内容更新时间。

推荐增强规则：

- 如果模板已经被历史 run 使用，再编辑时提示用户：

```text
该模板已被历史任务使用。继续修改不会影响历史任务，因为任务已保存提示词快照。
```

后续可升级为：

- 已使用模板不可直接编辑。
- 编辑时自动复制为新版本，例如「我的模板 v2」。

第一版暂不强制自动版本化，但 UI 文案中应清楚说明运行快照规则。

## 与现有标注流水线2的关系

### 当前逻辑

当前 `professional_cot_service.py` 直接从 `PROMPT_ROOT` 读取提示词文件。

### 改造后逻辑

需要抽出提示词读取服务，例如：

```text
professional_cot_prompt_service.py
```

职责：

- 初始化系统默认模板。
- 从 md 文件构建系统默认模板结构。
- 管理用户模板文件。
- 根据 `template_id` 和 `prompt_key` 读取 Prompt。
- 创建 run 时写入完整快照。
- 运行过程中优先从 run 快照读取 Prompt。

标注流水线2执行步骤不再直接调用 `_extract_step_prompt()` / `_type_prompt()` 读取仓库 md，而是通过模板服务读取当前 run 快照中的提示词。

## 兼容规则

1. 若旧 run 没有 `prompt_template` 字段，详情页显示「旧版本任务，未记录提示词模板」。
2. 若系统模板目录不存在，后端可从 `docs/background/3类COT提示词/` 自动初始化。
3. 若用户指定模板不存在或无权限，创建任务失败，不应静默切换到其他模板。
4. 若运行中快照文件缺失，任务应失败并记录错误，避免使用错误模板继续生成。
5. 批量文献 run 中，若单篇文献处理失败，不应重新解析或切换提示词模板；后续文献继续使用同一个 run 快照。

## 验收标准

1. COT/H-COT 下可进入「标注流水线2提示词」管理页。
2. 页面可以查看系统默认模板，但不能编辑系统默认模板。
3. 用户可以复制系统默认模板为自己的模板。
4. 用户可以在自己的模板中编辑任意一个 Prompt 并保存。
5. 左侧按「通用步骤 + CoT 类型 + Step」树形展示，不平铺 33 条。
6. 新建标注流水线2任务时可以选择提示词模板版本。
7. 未设置个人默认模板时，新建任务默认选系统默认模板。
8. 设置个人默认模板后，新建任务默认选该模板。
9. 任务启动后，run 目录下生成完整提示词快照。
10. 运行详情页展示本次使用的模板名称和快照信息。
11. 修改用户模板后，不影响历史 run 的提示词快照。
