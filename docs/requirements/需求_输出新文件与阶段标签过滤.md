# 需求：输出新文件不覆盖 + 阶段标签过滤

## 背景

当前 Pipeline 的写回机制存在两个使用上的问题：

1. **覆盖原文件**：除 `question_generate` 外，其余 8 个阶段执行完毕后调用 `write_datasets_to_file()`，会把结果**直接覆盖到输入文件**（同一个 `file_id` 的 JSON）。一旦中间某阶段出错或想回退到上一阶段重跑，原始数据已经丢失。
2. **文件来源不清晰**：`File` 表已有 `source_stage` 字段，但前端选文件下拉不做按阶段过滤，所有 JSON 文件混在一起，用户在「答案生成」阶段也能看到原始上传文件、问题校验失败文件等无关项，容易选错。

本需求解决以上两点，作为分支 2 的工作内容。
分支：`feature/no-overwrite-and-stage-tag`

> 依赖：分支 1 (`feature/llm-task-improvements`) 必须先合并，因为本需求会复用分支 1 中已扩展的 `create_output_file(stage=...)` 和 `STAGE_LABELS`。

---

## 需求 4：每个阶段产出新文件，原文件保留

### 总原则
- **任何阶段不再覆盖输入文件**。
- 每次任务执行成功，都创建一个新的 `File` 记录（带 `source_stage` 标签）+ 物理文件，所有结果数据写入新文件。
- 输入文件完全不动，包括 DB 记录和磁盘文件。

### 文件命名
沿用分支 1 的命名规则：
```
{源文件名（去扩展）}_{当前阶段中文}_{username}_{YYYYMMDDHHmmss}.json
```

### Dataset 数据流转
关键改动：阶段执行时，不再原地修改输入文件对应的 Dataset 记录，而是：

1. 读输入文件的 Dataset 列表（按 `file_id` 查）。
2. 对每条记录调用 LLM 得到本阶段产物（如答案、评分、CoT 等）。
3. **复制（克隆）一份新的 Dataset 记录**，把本阶段新增/更新的字段写到新记录上，`file_id` 指向新建的输出 File，`current_stage` 设置为本阶段。
4. 原 Dataset 记录保持不变。

> 这种「每阶段克隆一次 Dataset」的语义保证了任意阶段都可以独立回溯、独立重跑。

### 校验阶段（保留 fail file 旧逻辑）
- `question_validate` / `answer_validate` 仍按现有方式产出 `_xxx校验失败.json`（`create_fail_file`）—— 该文件保留不变。
- 通过的记录走新的「输出新文件」路径，写入新 `File`（`source_stage = 当前校验阶段`），不再覆盖输入文件。

### 后处理阶段产出多文件
- **数据集切分**：train / test 各创建一个独立 `File` 记录，命名分别后缀 `_train` / `_test`，例：
  ```
  corpus_数据集切分_alice_20260505143012_train.json
  corpus_数据集切分_alice_20260505143012_test.json
  ```
  两份都带 `source_stage = DATASET_SPLIT`。
- **COT 过滤**：现有「保留 / 过滤掉」两份输出各自独立 `File` 记录，命名同上规则。
- **评分标准生成**：单文件，正常规则。

### file_service.py 重构
- `create_output_file()` 通用化：参数 `stage: StageEnum` 必填，去掉 `question_generate` 专用语义。
- 新增 `create_output_file_with_suffix()` 或可选 `name_suffix` 参数支持 train/test/keep/drop 这种附加后缀。
- `write_datasets_to_file()` 保留功能（仍是「按 file_id 把 Dataset 序列化写盘」），但调用时传入的是**新建的输出 file_id**，不再是源 file_id。
- 原代码注释「overwrite the JSON on disk」更新为「write the JSON to the output file（never overwrites input）」。

### 涉及阶段（全部改造）
| 阶段 | 当前写回方式 | 改造后 |
|---|---|---|
| 问题生成 | 已经是新文件（`create_output_file`） | 命名规则统一即可 |
| 知识体系 | 覆盖原文件 | 新文件 |
| 问题校验 | 覆盖原文件 + fail file | 新文件 + fail file（保留） |
| 答案生成 | 覆盖原文件 | 新文件 |
| 答案校验 | 覆盖原文件 + fail file | 新文件 + fail file（保留） |
| 数据评估 | 覆盖原文件 | 新文件 |
| COT 过滤 | 覆盖 / 多文件 | 多文件（独立 File） |
| 数据集切分 | 多文件 | 多文件（独立 File，每份带 source_stage） |
| 评分标准 | 覆盖原文件 | 新文件 |

### 验收
- 跑「答案生成」后：源文件磁盘内容、DB Dataset 记录都未变；新增 1 个 `File` 记录（`source_stage=ANSWER_GENERATE`）和对应磁盘文件，包含克隆出的 Dataset 数据。
- 跑「问题校验」后：源文件不变；新增 1 个通过文件 + 1 个 `_问题校验失败.json`。
- 跑「数据集切分」后：源文件不变；新增 train + test 两个独立 File 记录。
- 文件管理页能看到所有新生成文件，每个文件都带阶段标签。

## 需求 5：File 阶段 tag 过滤选文件下拉

### 上一阶段映射表（已确认）
| 当前阶段 | 默认展示来源 source_stage |
|---|---|
| 问题生成 | （全部 JSON，无过滤） |
| 知识体系 | QUESTION_GENERATE |
| 问题校验 | KNOWLEDGE_GENERATE |
| 答案生成 | QUESTION_VALIDATE |
| 答案校验 | ANSWER_GENERATE |
| 数据评估 | ANSWER_VALIDATE |
| COT 过滤 | DATA_EVALUATE |
| 数据集切分 | COT_FILTER |
| 评分标准生成 | DATASET_SPLIT |

### 后端改动
各阶段 `/source-files` 接口新增 `stage` 查询参数（可选，预留未来扩展）：
```python
@router.get("/source-files")
async def list_source_files(
    show_all: bool = False,
    current_user = Depends(get_current_user),
    db = Depends(get_db),
):
    q = db.query(File).filter(
        File.user_id == current_user.id,
        File.file_type == "json",
    )
    if not show_all:
        q = q.filter(File.source_stage == <本阶段对应的上一阶段>)
    return [...]
```

> 「问题生成」阶段的 `/source-files` 不做 stage 过滤（保持当前行为，但 `show_all` 参数仍接收，仅用于一致性）。

### 前端改动
- `FileSelector.vue` 新增 props：
  - `expectedStage`: 期望的来源 stage（如 `'question_generate'`）
  - 内部新增 `showAll` 开关（el-switch），默认 false。
  - 开关变化时重新拉取 `/source-files?show_all=<bool>`。
- 各阶段页面调用 `FileSelector` 时传入对应的 `expectedStage`。
- 「问题生成」页面 `expectedStage = null`，不显示开关。
- 开关 UI 文案：「显示所有阶段文件」（关闭状态）/「仅显示上一阶段文件」（开启状态）。建议统一为开关 + 标签：
  ```
  ☑ 仅显示上一阶段（问题校验）文件
  ```
  默认勾选；取消勾选则展示全部。

### 验收
- 进入「答案生成」页面：默认下拉只显示带 `source_stage=question_validate` 的文件。
- 取消勾选后下拉显示该用户所有 JSON 文件。
- 「问题生成」页面无开关，下拉显示所有 JSON 文件（与现状一致）。
- 用户上传新文件后，该文件 `source_stage=NULL`，默认不出现在过滤后的下拉中（除非在「问题生成」阶段或开关展开）。

## 涉及文件总览

后端：
- `backend/app/services/file_service.py`（核心重构：通用 create_output_file、移除覆盖语义）
- `backend/app/routers/*.py` 8 个非 question_generate 阶段（改写回逻辑）
- 各阶段 `/source-files` 接口加 `show_all` 过滤参数

前端：
- `frontend/src/components/FileSelector.vue`（加 expectedStage prop + 开关）
- `frontend/src/views/*.vue` 9 个阶段页面（传入 expectedStage）
- `frontend/src/api/index.js`（source-files API 加 show_all 参数）

数据库：
- 不需要新建表或字段，`File.source_stage` 字段已存在。
- 历史已有文件 `source_stage` 可能为 NULL，无需迁移；用户可手动改用开关查看。
