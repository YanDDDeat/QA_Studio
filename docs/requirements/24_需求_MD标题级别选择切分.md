# 24_需求_MD标题级别选择切分

## 背景

当前 MD 上传转换支持“整篇不切分 / 按章节切分 / 按段落切分”。其中“按章节切分”需要用户手动填写最小标题层级和最大标题层级，用户在未查看 MD 标题结构时难以判断应按几级标题切分。

本需求要求：用户选择 MD 文件后，系统先读取并解析该 MD 中实际存在的标题层级，展示可选层级及数量；用户选择某一级标题后，再按该级标题作为 chunk 边界进行切分。

## 输入定义

### 前端输入

- 用户在 FileSelector 上传区域选择 `.md` 文件。
- 来源类型：`文献` / `图书` / `其他`。
- 转换策略：
  - `整篇不切分`
  - `按指定标题级别切分`
  - `按段落切分`
- 当选择“按指定标题级别切分”时：
  - 前端先解析标题预览。
  - 用户只能从当前 MD 实际存在的标题层级中选择一个标题级别。

### 后端输入

新增标题预览接口接收 multipart form：

- `file`: `.md` 文件。

MD 上传转换接口扩展参数：

- `split_mode`: 保留现有值，并新增或复用章节切分模式。
- `heading_level`: 可选，`1~6`，表示按指定标题级别切分。
- `source_type`: `文献` / `图书` / `其他`。
- `min_chars`: 段落切分最小字符数，保持现有逻辑。

## 输出定义

### 标题预览接口输出

返回当前 MD 中识别到的 Markdown 标题：

```json
{
  "headings": [
    { "level": 1, "title": "绪论", "line": 1 },
    { "level": 2, "title": "研究背景", "line": 8 }
  ],
  "available_levels": [1, 2],
  "level_counts": {
    "1": 1,
    "2": 1
  }
}
```

### 上传转换输出

保持现有 `/file-manage/upload-md` 返回结构：

```json
{
  "uploaded": [
    {
      "id": 1,
      "filename": "xxx.json",
      "file_type": "json",
      "text_field": "text"
    }
  ],
  "errors": []
}
```

生成 JSON 记录继续保持：

```json
{
  "text": "章节完整文本",
  "source": "原文件名_序号",
  "source_type": "图书",
  "source_id": "原文件名_序号"
}
```

## 详细规则

### 标题识别规则

- 识别标准 Markdown ATX 标题：`#` 到 `######`。
- 标题格式必须为：行首 `1~6` 个 `#`，后跟至少一个空白字符，再跟标题文本。
- 示例：
  - `# 一级标题`：识别。
  - `## 二级标题`：识别。
  - `#标题`：不识别。

### 标题预览规则

- 用户选择 MD 文件后，前端调用标题预览接口。
- 后端只解析标题，不保存 MD，不生成 JSON 文件，不写数据库。
- 前端展示实际存在的标题层级及数量。
- 如果未识别到标题，提示用户无法按标题切分，可选择整篇不切分或按段落切分。

### 按指定标题级别切分规则

选择标题级别 N 后：

- 仅以 N 级标题作为 chunk 起点和边界。
- N 级标题下方的更低级标题（如 N+1、N+2）及其正文归入当前 chunk。
- 遇到下一个 N 级标题时，结束当前 chunk，开始新 chunk。
- N 级标题之前的内容不单独生成 chunk。
- 更高级标题（小于 N 的标题）默认不作为 chunk 边界；如果它出现在两个 N 级标题之间，会作为当前内容的一部分处理。

示例：选择按 2 级标题切分：

```md
# 第一章
章简介

## 1.1 背景
正文 A

### 1.1.1 子节
正文 B

## 1.2 方法
正文 C
```

应生成 2 个 chunk：

```text
chunk 1:
## 1.1 背景
正文 A
### 1.1.1 子节
正文 B

chunk 2:
## 1.2 方法
正文 C
```

## UI / 接口变更说明

### 前端变更

- `FileSelector.vue` 在选择 MD 文件后自动解析标题。
- “按章节切分”改为更明确的“按标题切分”。
- 当选择“按标题切分”时：
  - 展示标题解析状态。
  - 展示可选标题层级，例如：`一级标题（3个）`、`二级标题（12个）`。
  - 用户必须选择一个实际存在的标题层级后才能上传。
- 保留“整篇不切分”和“按段落切分”。

### 后端变更

- 新增 MD 标题预览接口。
- 扩展 MD parser：
  - 提供标题扫描函数。
  - 提供按指定标题级别切分函数。
- `/file-manage/upload-md` 支持 `heading_level` 参数。
- 保持现有 JSON 输出格式和文件管理记录逻辑不变。

## 非目标

- 不做数据库结构变更。
- 不保存原始 MD 文件。
- 不实现 Markdown AST 级复杂解析。
- 不处理 Setext 标题（`===` / `---`）。
