# 36_需求_CoTHCoT 文本预处理批量上传与单 COT 文件过滤

## 背景

CoT/H-CoT 标注流程中，文本预处理和单 COT 生成依赖数据中心内的 JSON 文件作为输入。

当前存在两个使用痛点：

1. 「CoT/H-CoT 标注 -> 文本预处理 -> 大文献（博士论文）拆分及预处理」上传 Markdown 时一次只能选择一个 MD 文件。博士论文或大型文献经常按文件拆分整理，逐个上传、逐个选择拆分策略成本较高。
2. 「CoT/H-CoT -> 单COT生成 -> 新建流水线 -> 系统 JSON 文件」选择已有 JSON 时默认展示系统内全部 JSON 文件，包括流水线中间产物和下游结果文件，文件数量较多时难以定位上传阶段的原始输入文件。

本需求优化这两个入口，提高批量准备数据和选择源文件的效率。

## 输入输出定义

### 输入

#### 批量 MD 上传

用户在「大文献（博士论文）拆分及预处理」的上传入口中选择一个或多个 `.md` 文件，并配置同一套转换参数：

- 来源类型：`文献` / `图书` / `其他`。
- 转换策略：
  - 整篇不切分；
  - 按 Markdown 标题层级；
  - 按编号层级；
  - 按段落切分。
- 策略参数：标题层级、编号深度、段落最小字符数等。

#### 单 COT 源文件选择

用户在「单COT生成」新建流水线弹窗中选择系统已有 JSON 文件。

### 输出

#### 批量 MD 上传

- 每个 MD 文件独立转换并保存为一个 JSON 文件。
- 例如：`论文A.md -> 论文A.json`，`论文B.md -> 论文B.json`。
- 输出 JSON 保持现有记录结构：

```json
[
  {
    "text": "...",
    "source": "...",
    "source_type": "图书",
    "source_id": "..."
  }
]
```

- 成功文件进入数据中心，`file_type=json`，`text_field=text`，`source_stage` 为空，表示上传阶段源文件。
- 上传完成后前端刷新文件列表，并选择本次成功上传的第一个文件作为当前文件。

#### 单 COT 源文件选择

- 默认只展示上传阶段 JSON 文件，即 `source_stage IS NULL` 的文件。
- 包括手动上传 JSON、MD 转 JSON、多 MD 合并保存到数据中心的 JSON。
- 不默认展示流水线中间产物或后续阶段输出文件。

## 详细规则

### 批量 MD 上传规则

1. 上传控件允许一次选择或拖拽多个 `.md` 文件。
2. 多文件共用同一套来源类型和拆分策略，不为每个文件单独配置策略。
3. 后端已有 `/api/file-manage/upload-md` 多文件能力，应优先复用现有接口。
4. 每个 MD 文件独立生成一个 JSON 文件，某个文件失败不应阻断其它文件保存。
5. 前端应展示或提示成功数量与失败明细；失败信息应包含文件名和错误原因。
6. 上传成功后清空上传队列，刷新可选文件列表，并切回「选择已有文件」模式。
7. 如果多个 MD 文件中标题结构不一致，标题层级预览仅作为策略辅助，不强制逐文件独立配置；实际解析失败的文件进入失败明细。
8. JSON 上传仍保持单文件或现有行为，不因本需求扩大为多 JSON 批量上传。

### 单 COT 源文件过滤规则

1. 「单COT生成」新建流水线弹窗默认请求文件列表时，应带上传阶段过滤条件。
2. 过滤条件使用现有文件管理接口支持的 `source_stage=upload`，即查询 `File.source_stage IS NULL`。
3. 前端仍只接受 JSON 文件作为系统已有文件选项。
4. 下拉框保留 `filterable` 搜索能力，方便按文件名查找。
5. 如保留「显示全部文件」能力，应默认关闭；本需求最低要求是默认只展示上传阶段文件。

## UI / 接口变更说明

### 前端

- 修改 `frontend/src/components/FileSelector.vue`：
  - MD 上传时支持多文件选择；
  - 上传提交时将所有 MD 文件追加到 `FormData.files`；
  - 多文件成功/失败提示更清晰；
  - 上传成功后刷新并选中第一个成功文件。
- 修改 `frontend/src/views/CotHcot/ProfessionalCotList.vue` 或相关 API 调用：
  - 获取单 COT 源文件列表时默认传入 `source_stage=upload`；
  - 继续过滤 `file_type=json`。

### 后端

- 优先不新增接口，不变更数据库。
- 复用 `backend/app/routers/file_manage.py` 中已有 `/upload-md` 多文件处理能力。

## 验收标准

1. 在大文献预处理上传入口一次选择 2 个或以上 MD 文件，点击上传后，每个成功文件都在数据中心生成独立 JSON。
2. 批量上传中某个 MD 解析失败时，前端能提示该文件失败原因，其他成功文件仍保留。
3. 上传成功后，预处理页面的文件选择列表刷新，并默认选中第一个成功上传的 JSON 文件。
4. 单 COT 生成新建流水线弹窗中，「系统 JSON 文件」默认只显示上传阶段 JSON 文件。
5. 单 COT 文件下拉不再默认混入 `generic`、`cot_hcot_pipeline`、`quality_check` 等阶段产物。
6. 不新增数据库表或字段，不需要迁移脚本。
