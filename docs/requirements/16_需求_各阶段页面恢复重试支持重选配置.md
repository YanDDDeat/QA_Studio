# 需求：各阶段页面恢复/重试按钮支持重选配置

## 背景

「我的任务」页面（`frontend/src/views/MyTasks.vue`）的恢复/重试按钮点击后会弹窗，让用户重新选择 prompt、model、llm_config（厂商），然后把新配置传给后端覆盖原任务设置。

但 7 个阶段页面（QuestionGenerate / KnowledgeGenerate / QuestionValidate / AnswerGenerate / AnswerValidate / DataEvaluate / CotFilter / DatasetProcessing）的恢复/重试按钮**直接调用 `resumeTask(taskId)` / `retryStage(slug, taskId)`，不传第二参数**，导致沿用原任务配置，无法切换。

行为不一致让用户产生困惑：在哪儿点恢复决定能不能换配置。

## 目标

7 个阶段页面的恢复/重试按钮行为对齐 MyTasks.vue：点击后弹窗，让用户重新选 prompt / model / llm_config，确认后再调接口。

## 范围

### 涉及页面（共 8 个）

| 文件 | 按钮 | 调用 |
|---|---|---|
| `QuestionGenerate.vue` | 恢复 / 重试 | resumeTask / retryStage('question-generate', ...) |
| `KnowledgeGenerate.vue` | 恢复 / 重试 | resumeTask / retryStage('knowledge-generate', ...) |
| `QuestionValidate.vue` | 恢复 / 重试 | resumeTask / retryStage('question-validate', ...) |
| `AnswerGenerate.vue` | 恢复 / 重试 | resumeTask / retryStage('answer-generate', ...) |
| `AnswerValidate.vue` | 恢复 / 重试 | resumeTask / retryStage('answer-validate', ...) |
| `DataEvaluate.vue` | 恢复 / 重试 | resumeTask / retryStage('data-evaluate', ...) |
| `CotFilter.vue` | 恢复（无重试） | resumeTask |
| `DatasetProcessing.vue` | 恢复（split + assess 两个任务，无重试） | resumeTask |

### 抽取公共组件

新建 `frontend/src/components/TaskConfigDialog.vue`，组件内置：
- 三个下拉：厂商（LLMConfig）、模型（依赖厂商，从该 LLMConfig.models 列出）、提示词（按当前 stage 过滤的 Prompt 列表）
- 默认值：传入 task 对象时，自动从 `task.prompt_id` / `task.model` / `task.llm_config_id`（或从 prompt 关联取）回填，让用户在原配置基础上修改
- 「取消」「确认」按钮
- emit `confirm({ prompt_id?, model?, llm_config_id? })` 事件，未改动的字段不传

各阶段页面 import 该组件，点击恢复/重试时先打开 dialog，dialog confirm 后再调 `resumeTask(taskId, data)` / `retryStage(slug, taskId, data)`。

### MyTasks.vue 同步改造

把现在 MyTasks.vue 里的内联弹窗逻辑也替换成 `TaskConfigDialog.vue`，避免双份维护。

## 后端

无改动。`resumeTask` 和 `retryStage` 接口已经支持第二参数 `{ prompt_id?, model?, llm_config_id? }`（参考 MyTasks.vue 的现有实现）。

## 测试方案

人工：
- 启动一个 question_generate 任务、暂停、点恢复 → 应弹窗 → 改 prompt → 确认 → 后端 task 的 prompt_id 应变成新值
- 同上换成 retry 路径，校验同样行为
- 校验 8 个页面行为一致
- CotFilter / DatasetProcessing 只有 resume，没有 retry，按钮数量不变

## 不做的事

- 不改后端
- 不动 MyTasks 表格 UI（只替换弹窗组件）
- 不改 stop 按钮逻辑
