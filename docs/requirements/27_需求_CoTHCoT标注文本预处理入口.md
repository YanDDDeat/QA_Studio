# 需求 27：CoT/H-CoT 标注分组新增文本预处理入口

## 背景

当前系统已有「文本预处理」页面，但入口位于主流程相关位置。为了在 CoT/H-CoT 标注工作流中更方便地对论文/语料文件进行预处理，需要在侧边栏「CoT/H-CoT 标注」分组下新增一个文本预处理入口。

## 需求说明

在现有「CoT/H-CoT 标注」侧边栏分组下，新增一个菜单项，页面内容复用或复制现有 `frontend/src/views/TextPreprocess.vue` 的能力。

建议菜单名称：

```text
文本预处理
```

建议内部路由：

```text
/cot-hcot-text-preprocess
```

建议组件实现方式：

- 优先复用现有 `TextPreprocess.vue`，避免复制导致后续维护两份逻辑。
- 如果必须复制，也应保持功能、交互和样式与原页面一致。

## 多 MD 文件合并为 JSON

在 CoT/H-CoT 标注分组下的「文本预处理」页面中，需要额外提供一个“多 MD 合并 JSON”功能，用于把多个 Markdown 文件整理成后续标注流水线可选择的 JSON 文件。

### 输入

- 用户可以一次上传多个 `.md` 文件。
- 每个文件读取完整 Markdown 文本内容。
- 文件名作为 `source` 字段。

### 输出

将多个 MD 文件合并为一个 JSON 数组：

```json
[
  {
    "source": "文件1.md",
    "text": "文件1的 Markdown 全文内容"
  },
  {
    "source": "文件2.md",
    "text": "文件2的 Markdown 全文内容"
  }
]
```

规则：

- 输出字段固定为 `source` 和 `text`。
- `source` 使用原始文件名。
- `text` 使用 MD 文件全文内容。
- 输出 JSON 应支持下载。
- 若可复用现有文件上传/文件管理能力，也可以在生成后保存为系统已有 JSON 文件，便于后续「标注流水线2」直接选择。
- 第一版不需要对 MD 内容做切分、清洗、标题识别或 token 过滤，只做多文件合并。

## 范围

本需求增加 CoT/H-CoT 标注分组下的文本预处理入口，并在该入口提供多 MD 文件合并为 JSON 的辅助功能。不新增数据库表/字段，不新增迁移。

## 验收标准

1. 侧边栏「CoT/H-CoT 标注」分组下出现「文本预处理」菜单项。
2. 点击该菜单后进入文本预处理页面。
3. 页面功能与现有「文本预处理」页面保持一致。
4. 原有「文本预处理」入口不受影响。
5. CoT/H-CoT 标注分组下的文本预处理页面支持一次上传多个 MD 文件。
6. 上传多个 MD 后可生成 JSON 数组，每个元素包含 `source` 和 `text`。
7. `source` 等于原始文件名，`text` 等于该 MD 文件全文内容。
8. 生成的 JSON 支持下载；如实现保存到系统文件，也应能被后续标注流水线2选择。
9. 不产生数据库迁移。
