# CoT / H-CoT 标注说明

## 0. 这份说明解决什么问题

这份说明用于完成两类数据：

1. **博士论文 H-CoT 数据**：构建 L0 / L1 / L2 多层级问题树，每个层级都有完整的 `问题-答案-思维链`。
2. **研究论文 CoT 数据**：不构建层级树，只构建多个彼此独立的 `问题-答案-思维链`。

只需要按照步骤阅读文献、抽取规律、填写 JSON 模板即可。

---

## 1. 标注总原则

### 1.1 标注目标

我们不是做普通问答，也不是摘抄论文结论，而是构建能够训练模型“分析原因、解释机制、形成推理”的数据。

每条数据都要包含：

```json
{
  "input": "问题",
  "output": "完整答案",
  "chainofThought": "思维链",
  "level": "L0/L1/L2，仅博士论文 H-CoT 需要"
}
```

研究论文 CoT 不需要 `level` 字段。

### 1.2 问题类型以机理分析为主

优先标注这类问题：

- 为什么某种结构能提升性能？
- 为什么某种组分会促进或抑制反应？
- 为什么某种工艺能改变形貌或界面？
- 为什么微观结构变化会影响宏观应用性能？
- 为什么某种材料有优势但仍存在应用边界？

不要优先标注这类问题：

- 某个实验数值是多少？
- 某个样品提升了百分之多少？
- 某张图说明了什么？
- 某章节研究了什么？

### 1.3 不能突兀出现论文实验数值

最终的 `问题`、`答案`、`思维链` 中不要直接写来源文献里的具体实验数值。

错误写法：

```text
当 Al 含量为 25% 时，峰值压力达到 7.33 MPa，因此反应性能最好。
```

推荐写法：

```text
当金属燃料比例与氧化剂供氧能力匹配时，氧化还原反应更充分，能量释放效率更高。
```

可以使用方向性表达：

- 显著提升
- 明显降低
- 更均匀
- 更稳定
- 更有利于传热传质
- 反应更充分
- 安全性改善
- 加工性增强

### 1.4 不要写来源提示语和节点提示语

不能在答案或思维链中写：

```text
证据摘要显示……
论文表明……
本文研究发现……
该事实卡说明……
L2-1-1 说明……
该 L2 答案支撑 L1……
综合上述 L1……
```

推荐直接写客观规律：

```text
核壳结构能够缩短氧化剂与金属燃料之间的传质距离，使界面反应更充分。
```

## 2. 标注前准备

标注前先快速浏览文献，重点看：

1. 摘要
2. 每章小结
3. 结论
4. 创新点
5. 图表说明附近的总结性文字
6. 讨论机理、构效关系、组效关系的段落

不要从头到尾逐字精读。先抓住论文的核心逻辑：

```text
研究对象是什么？
做了哪些结构或组分设计？
解决了什么问题？
性能为什么会变化？
最终应用在哪里？
```

---

## 3. 第一步：抽取事实卡

事实卡不是训练数据，是后续写问题和答案的材料。

### 3.1 事实卡要写什么

每张事实卡只记录一个清楚的科学规律。

模板：

```json
{
  "fact_id": "F-0001",
  "topic": "主题",
  "object": "研究对象",
  "mechanism": "机理规律",
  "performance_implication": "性能意义",
  "evidence_ids": ["E-0001"],
  "raw_numeric_mentions": []
}
```

### 3.2 示例

```json
{
  "fact_id": "F-0001",
  "topic": "核壳结构界面反应",
  "object": "CL-20/Al 三组分含能微单元",
  "mechanism": "核壳结构使氧化剂与金属燃料在明确界面中均匀接触，减少纳米 Al 无序团聚并增强界面反应效率。",
  "performance_implication": "核壳结构能够提升反应活性、压力释放能力和燃烧稳定性。",
  "evidence_ids": ["E-0003", "E-0023"],
  "raw_numeric_mentions": ["原文中的压力、燃速、比例等具体数值"]
}
```

### 3.3 事实卡自查

合格事实卡应该满足：

- 是一个规律，不是一个孤立数值。
- 能回答“为什么”。
- 能用于后续生成问题。
- 没有编造论文不存在的内容。
- 如果原文有数值，只放到 `raw_numeric_mentions`，不要放进机理表达里。

---

## 4. 第二步：数值抽象

数值抽象是把事实卡中可能进入训练数据的内容改成不依赖具体数值的表达。

### 4.1 数值抽象模板

```json
{
  "fact_id": "F-0001",
  "topic": "核壳结构界面反应",
  "object": "CL-20/Al 三组分含能微单元",
  "mechanism_sanitized": "核壳结构使氧化剂与金属燃料在明确界面中均匀接触，减少纳米 Al 无序团聚并增强界面反应效率。",
  "performance_implication_sanitized": "核壳结构能够提升反应活性、压力释放能力和燃烧稳定性。",
  "numeric_dependency": "abstractable",
  "evidence_ids": ["E-0003", "E-0023"]
}
```

### 4.2 `numeric_dependency` 怎么填

填写三种之一：

```text
none：原本就不依赖具体数值。
abstractable：原文有数值，但可以改写成方向性规律。
numeric_dependency_required：离开具体数值后结论无法成立，通常不建议进入训练数据。
```

## 5. 博士论文 H-CoT 标注流程

博士论文需要构建层级问题树。

层级关系如下：

```text
L0：总问题，回答论文核心机制或总体结构-性能关系。
L1：子问题，分别回答 L0 的必要方面。
L2：子子问题，回答 L1 中更细的必要机制。
```

要求：

```text
L2 支撑 L1，L1 支撑 L0。
L0、L1、L2 都必须是独立问题。
每个层级都要有完整答案和思维链。
```

### 5.1 生成 L0 总问题

L0 是一个总问题，通常对应整篇博士论文的核心科学问题。

好问题示例：

```text
为什么多级微纳结构设计能够使 CL-20 基含能微单元同时改善能量释放效率、安全性、加工性和推进剂应用表现？
```

不合格问题：

```text
本文主要研究了什么？
```

```text
CL-20 微单元的燃速是多少？
```

### 5.2 将 L0 拆成 L1

L1 是回答 L0 必不可少的几个方面。

例如 L0 是：

```text
为什么多级微纳结构设计能够同时改善能量释放效率、安全性、加工性和应用表现？
```

可以拆成：

```text
L1-1：静电喷雾和分级组装为什么适合构筑结构可控的 CL-20 基含能微单元？
L1-2：黏结剂包覆和双组分微球结构为什么能够改善 CL-20 的安全性并保持能量潜力？
L1-3：核壳型三组分 CL-20/Al 微单元为什么能提高氧化剂与金属燃料的反应效率？
L1-4：石榴结构 CL-20 基含能微单元为什么能够同时提升传热传质、加工性和安全性？
L1-5：CL-20 基多级含能微单元为什么能改善丁羟推进剂的流变、力学和燃烧综合表现？
```

### 5.3 将 L1 拆成 L2

L2 是更细的独立机制问题。

例如 L1 是：

```text
核壳型三组分 CL-20/Al 微单元为什么能提高氧化剂与金属燃料的反应效率？
```

可以拆成：

```text
L2-3-1：核壳结构为什么能增强 CL-20 与纳米 Al 之间的界面反应效率？
L2-3-2：含能黏结剂为什么既可能促进三组分微单元释能，又可能在过量时抑制反应？
L2-3-3：为什么 CL-20/Al 微单元中金属燃料含量需要服从燃氧匹配而不能单纯增加？
```

### 5.4 写 L2 完整 CoT

L2 是最底层问题，要写清楚一个具体机制。

模板：

```json
{
  "id": "L2-3-1",
  "level": "L2",
  "input": "核壳结构为什么能增强 CL-20 与纳米 Al 之间的界面反应效率？",
  "output": "完整答案",
  "chainofThought": "完整思维链文本",
  "supports": "L1-3"
}
```

写作要求：

- `output` 至少写清楚“原因 + 过程 + 结果”。
- `chainofThought` 写客观推理规律，不写“该 L2 支撑 L1”。
- 不要出现具体实验数值。

### 5.5 写 L1 完整 CoT

L1 的答案必须自然包含下属 L2 的关键内容。

也就是说，L1 不能只写一句总结，必须把它下面几个 L2 的机制融合进去。

错误写法：

```text
核壳结构可以提高反应效率，因此三组分微单元性能更好。
```

推荐写法：

```text
核壳型三组分微单元能够提高反应效率，是因为它同时优化了界面接触、黏结剂协同和燃氧匹配。核壳结构使 CL-20 与纳米 Al 在连续界面附近均匀接触，减少无序团聚并缩短传热传质距离；含能黏结剂在适量时参与放热并稳定结构，但过量时会阻碍反应通道；金属燃料比例也需要与氧化剂供氧能力匹配，避免反应不完全。
```

### 5.6 写 L0 完整 CoT

L0 的答案必须自然包含所有 L1 的关键内容，并适当吸收重要 L2 机制。

L0 不是摘要，不是结论复述，而是总机制解释。

L0 答案一般需要覆盖：

- 制备方法为什么能控制结构；
- 双组分结构为什么能改善安全性；
- 三组分核壳结构为什么能增强反应；
- 石榴结构为什么能改善传热传质和加工性；
- 推进剂应用中为什么能改善流变、力学和燃烧。

## 6. 研究论文 CoT 标注流程

研究论文不做 L0/L1/L2 层级树，只做多个独立 CoT。

### 6.1 什么时候用研究论文 CoT

适用于：

- 单篇期刊论文；
- 篇幅较短；
- 主题集中；
- 不适合拆成完整多层级问题树。

### 6.2 标注步骤

研究论文 CoT 的步骤：

```text
阅读摘要、结论、讨论段落
抽取事实卡
做数值抽象
生成多个独立问题
逐条写完整答案和思维链
自查格式和内容
```

### 6.3 独立问题怎么写

每个问题都应该脱离其他问题也能回答。

好问题：

```text
氧平衡为什么适合作为绿色氧化剂初筛指标，但不能单独决定候选物是否适合固体推进剂应用？
```

不好问题：

```text
上述结果说明了什么？
```

```text
表 2 中哪种氧化剂性能最好？
```

### 6.4 研究论文 CoT 模板

```json
{
  "sample_id": "RP_COT_0001",
  "input": "问题",
  "output": "完整答案",
  "chainofThought": "思维链文本",
  "metadata": {
    "source_fact_ids": ["F-0001"],
    "numeric_policy": "sanitized"
  }
}
```

研究论文最终训练数据只需要：

```json
{
  "input": "问题",
  "output": "完整答案",
  "chainofThought": "思维链文本"
}
```

---

## 7. 答案怎么写

答案要做到：

```text
先回答结论，再解释原因，再说明边界或意义。
```

推荐结构：

```text
某结构/组分/方法能够产生某种效果，核心原因是……。
首先，……。
其次，……。
同时，……。
因此，它能够……，但也需要注意……。
```

答案不要太短。  
如果一个问题涉及多个机制，答案必须把多个机制都写出来。

---

## 8. 思维链怎么写

这里的思维链不是随便列关键词，而是把答案背后的推理过程写成客观规律总结。

### 8.1 思维链要求

思维链应该做到：

- 是完整文本，不是项目符号列表。
- 不写来源文献提示语。
- 不写节点支撑提示语。
- 不写具体实验数值。
- 不写“我认为”“可以看出”等主观话。
- 按因果顺序展开。

### 8.2 推荐写法

```text
推进剂氧化剂的实际价值来自完整体系表现，而不是单个分子参数。高能特征能够提高潜在性能上限，但吸湿、敏感性、相容性和加工困难会影响安全、储存和燃烧稳定。不同指标对应不同应用环节，供氧能力影响燃烧，稳定性影响储存和使用，界面相容性影响加工与力学完整性。因此，多指标综合筛选能把候选物从分子潜力推进到配方可用性判断。
```

### 8.3 不推荐写法

```text
第一，论文表明……
第二，L2-1-1 说明……
第三，实验数据为……
因此，该答案支撑 L1……
```

## 9. 最终自查清单

每条样本完成后，逐条检查：

### 9.1 问题检查

- 问题是否独立？
- 是否是机理分析、方法作用、结构-性能关系或应用边界问题？
- 是否没有“本文、该论文、上述、这个实验”等依赖上下文的词？
- 是否没有具体实验数值？

### 9.2 答案检查

- 是否完整回答了问题？
- 是否解释了原因、过程和结果？
- 是否没有直接复述论文数值？
- 是否没有“证据显示、论文表明”等表达？
- H-CoT 中父节点答案是否自然包含子节点关键内容？

### 9.3 思维链检查

- 是否是拼接好的文本？
- 是否按因果逻辑展开？
- 是否没有列表编号或节点编号元话语？
- 是否没有具体实验数值？
- 是否不是简单重复答案？

### 9.4 H-CoT 层级检查

只针对博士论文：

- L0 是否是总问题？
- L1 是否分别回答 L0 的必要方面？
- L2 是否分别支撑 L1 的具体机制？
- L0、L1、L2 是否都能独立作为训练样本？
- L1 答案是否包含 L2 关键机制？
- L0 答案是否包含 L1 和关键 L2 内容？

## 10. 常见错误

### 错误 1：把实验数值写进答案

错误：

```text
样品的峰值压力达到某具体数值，所以性能最好。
```

正确：

```text
样品表现出更强的压力释放能力，说明结构设计促进了反应物接触和能量释放。
```

### 错误 2：问题不独立

错误：

```text
这种结构为什么更好？
```

正确：

```text
石榴结构 CL-20 基含能微单元为什么能够改善传热传质和燃烧反应效率？
```

### 错误 3：答案太短

错误：

```text
因为核壳结构可以增强界面反应。
```

正确：

```text
核壳结构能够增强界面反应，是因为它使氧化剂和金属燃料在连续界面附近均匀接触，减少无序团聚，并缩短热量和反应物扩散距离。这样，氧化剂分解产生的热量和氧化性环境更容易作用于金属燃料，反应更充分，能量释放效率也更高。
```

### 错误 4：思维链写成节点说明

错误：

```text
L2-3-1 说明核壳结构能提高反应效率，因此支撑 L1-3。
```

正确：

```text
核壳结构将氧化剂和金属燃料组织在连续界面附近，减少无序团聚并缩短传热传质距离。界面接触越充分，氧化还原反应越容易进行，能量释放效率也越高。
```

## 11. 标注同学的实际工作顺序

### 博士论文 H-CoT

```text
1. 看摘要、各章小结、结论、创新点。
2. 抽取 8-15 张事实卡。
3. 将事实卡改写为去数值版本。
4. 写 1 个 L0 总问题。
5. 将 L0 拆成 3-6 个 L1 问题。
6. 每个 L1 拆成 2-4 个 L2 问题。
7. 先写 L2 的答案和思维链。
8. 再写 L1 的答案和思维链，吸收对应 L2 内容。
9. 最后写 L0 的答案和思维链，吸收所有 L1 和关键 L2 内容。
10. 按自查清单检查。
```

### 研究论文 CoT

```text
1. 看摘要、引言末段、结果讨论、结论。
2. 抽取 5-10 张事实卡。
3. 将事实卡改写为去数值版本。
4. 生成 5-10 个独立问题。
5. 逐条写答案和思维链。
6. 按自查清单检查。
```

## 13. 一句话记忆

博士论文 H-CoT：

```text
先搭问题树，再从 L2 往 L0 写完整答案和思维链。
```

研究论文 CoT：

```text
不搭树，围绕论文核心规律写多个独立问题和完整 CoT。
```

共同要求：

```text
写机制，不写数值；写规律，不写摘抄；写完整推理，不写节点说明。
```

## 14. 如何借助大模型完成标注

标注同学不需要完全手工写所有内容，可以把论文片段、证据块或前一步结果复制给大模型，让大模型先生成初稿，再由人工检查和修改。

使用大模型时要注意：

